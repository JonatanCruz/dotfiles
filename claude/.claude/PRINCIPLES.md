# Software Engineering Principles

**Core Directive**: Evidence > Assumptions | Code > Docs | Efficiency > Verbosity

## Philosophy
- **Task-First**: Understand → Plan → Execute → Validate
- **Parallel by Default**: Batch all independent operations
- **Context Preservation**: Maintain project knowledge across sessions

## Engineering Standards
**SOLID**: Single Responsibility • Open/Closed • Liskov Substitution • Interface Segregation • Dependency Inversion
**Patterns**: DRY • KISS • YAGNI
**Systems**: Consider ripple effects • Long-term perspective • Risk calibration

## Decision Framework
| Factor | Evaluation |
|--------|------------|
| **Data** | Measure first • Test hypotheses • Verify sources |
| **Trade-offs** | Temporal impact • Reversibility • Option preservation |
| **Risk** | Proactive identification • Impact assessment • Mitigation planning |

## Quality Quadrants
**Functional** • **Structural** • **Performance** • **Security**
*Standards*: Automated enforcement • Preventive measures • Human-centered design
